🔄 Payroll Run Lifecycle

[1] Create
POST /api/hr/payroll/runs — Status: Draft (no items yet)
[2] Process
POST /api/hr/payroll/runs/{id}/process — يحسب كل الموظفين → Status: Processed
[3] Post
POST /api/hr/payroll/runs/{id}/post — يرحل للـ GL (Journal Entry) → Status: Posted
[4] Lock
Status: Posted — لا يمكن التعديل (للـ audit + compliance)

🏦 GL Posting (Smart CoA Fallback)

عند الـ Post، النظام ينشئ Journal Entry:

Debit  4200 G&A Expenses    | Salary + Social Insurance Employer
Credit 1210 Cash              | Net Pay + Employee Tax + Social Insurance Employee

إذا كان tenant عنده CoA مخصص (مثل 5500 بدل 4200)، النظام يستخدم smart fallback:

var expenseAccount = await coaService.GetByCodeAsync(tenantId, "4200", ct)
    ?? await coaService.GetByCodeAsync(tenantId, "5500", ct);

🐛 Bug Fixes (حرجة)

1. Dapper Enum Mapping

المشكلة: Dapper لا يحوّل string column إلى enum تلقائياً → 500 error.

الحل: إنشاء generic EnumStringTypeHandler<TEnum> + تسجيل 6 TypeHandlers:

SqlMapper.AddTypeHandler(new EnumStringTypeHandler<PayrollRunStatus>());
SqlMapper.AddTypeHandler(new EnumStringTypeHandler<PayrollItemStatus>());
SqlMapper.AddTypeHandler(new EnumStringTypeHandler<LeaveStatus>());
SqlMapper.AddTypeHandler(new EnumStringTypeHandler<PurchaseOrderStatus>());
SqlMapper.AddTypeHandler(new EnumStringTypeHandler<GoodsReceiptStatus>());
SqlMapper.AddTypeHandler(new EnumStringTypeHandler<VendorBillStatus>());

2. PayrollRepository SQL Syntax

المشكلة: PayrollRepository.GetItemsByRunAsync كان مفقود SELECT keyword.

// قبل (BUG):
var sql = $"{ItemSel} FROM payroll_items WHERE ...";  // ❌ 42601 syntax error

// بعد (FIX):
var sql = $"SELECT {ItemSel} FROM payroll_items WHERE ...";  // ✓

3. Redis 5s Timeout in Dev

المشكلة: StackExchange.Redis.PingAsync يحجب 5 ثواني عند عدم تشغيل Redis.

الحل: ConnectTimeout=1s, SyncTimeout=500ms, AsyncTimeout=500ms + CTS cap في HealthController.

النتيجة: /health/ready انتقل من 5000ms+ → 608ms ✅
