Bug #1 — Payroll Detail Page 500 Error FIXED

الخطورة: 🔴 High — المستخدم لا يستطيع رؤية تفاصيل دورة رواتب بعد الإنشاء

السبب: صفحة /hr/payroll/[id] كانت تُجرب SSR (Server-Side Rendering) — الـ JWT token يُخزّن في localStorage (client-side only) — الـ API call من الـ server يرجع 401 Unauthorized → Next.js throws 500 error

الحل:

// app/(authenticated)/hr/payroll/[id]/page.tsx export const dynamic = 'force-dynamic'; // ✅ منع SSR — الصفحة client-only الآن

النتيجة: ✅ POST /api/hr/payroll/runs → 201 → GET /api/hr/payroll/runs/{id} → 200

Bug #2 — Dashboard Sidebar API 404 Errors FIXED

الخطورة: 🟡 Medium — Dashboard widgets call /api/procurement/vendors و /api/procurement/pos و /api/hr/employees → كلهم 404 في الـ console

السبب: Dashboard تستخدم fetch('/api/procurement/vendors') — هذا يروح لـ localhost:3000 (frontend) بدل localhost:5000 (backend). الـ API endpoints موجودة في backend فقط.

الحل:

// dashboard/page.tsx — Before (❌) fetch('/api/procurement/vendors', { headers: { Authorization: ... } }) fetch('/api/procurement/pos', { ... }) fetch('/api/hr/employees', { ... }) // After (✅) — use typed procurementApi + hrApi procurementApi.listVendors(), // → localhost:5000/api/procurement/vendors procurementApi.listPOs(), // → localhost:5000/api/procurement/pos hrApi.listEmployees(), // → localhost:5000/api/hr/employees

النتيجة: ✅ Vendors: 200 (1 vendor) | POs: 200 | Employees: 200 (1 emp)

Bug #3 — Hydration Mismatch في Topbar FIXED

الخطورة: 🟡 Medium — 8 console errors من نوع "Text content did not match. Server vs Client" في كل صفحة

السبب: AppShell.tsx تستدعي authApi.getUser() مباشرةً أثناء render — في SSR يرجع null (لأن localStorage غير متاح) — في client يرجع بيانات حقيقية → Next.js hydration mismatch

// AppShell.tsx — Before (❌) const user = typeof window !== 'undefined' ? authApi.getUser() : null; const userName = user?.fullName || 'مستخدم'; // After (✅) — useState + useEffect (client-only) const [user, setUser] = useState(null); useEffect(() => { setUser(authApi.getUser()); }, []); const userName = user?.fullName || '';

النتيجة: ✅ Hydration errors eliminated
